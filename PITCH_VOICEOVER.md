Hi, I'm Dhruval. This summer I started learning pool. Every night I'd scroll Instagram and TikTok — hit it like this, use this cue for spin, try this drill — ten creators, ten opinions, none connected to the shot I'd just missed at the bar. I wasn't training. I was collecting tips with no feedback loop. Pool is hard because aiming is invisible. Pros see geometry: ghost ball, cut angle, tangent line. I saw two balls and a pocket and hope. I wanted structured training, not another reel selling a four-hundred-dollar cue. So I built Axiom — precision athletics where you aim a real shot, deterministic physics answers in milliseconds, and an optional multi-agent AI coach explains what you're seeing. Your stats track whether you're actually improving.

Architecture — three services, one contract. React + Vite + Tailwind frontend on port 3000: SVG billiards canvas, React Router, auth UI, training hub, PWA shell. Node.js Express + Socket.io broker on 3001: event broker, API gateway, JWT auth with bcrypt, Zod on every payload, Helmet, CORS, rate limiting, MongoDB with file-store fallback. Python 3.11 FastAPI + LangGraph engine on 8000: physics/core.py deterministic trig, graph.py async coach pipeline, httpx webhook telemetry. Flow: frontend emits compute_shot with cue/obj/pocket coords + with_coach flag → broker Zod-validates bounds (0-36 × 0-18) → POST /api/compute_shot or /api/analyze_shot → engine returns vector_data primitives + shot_solution numerics → each LangGraph node POSTs /api/webhook non-blocking → broker broadcasts agent_state_update → React agentReducer updates timeline, coach panel, narrative stream. Hard rule: LLMs narrate and assess risk. They never compute angles. coach_fallback.py survives OpenAI quota errors. Physics unit tests in physics.test_core.
